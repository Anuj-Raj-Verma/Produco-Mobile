Produco is a high-performance, proactive personal operating system. It was built to solve a critical flaw in standard habit trackers: *tracking data for the sake of tracking is a chore.* Instead of acting as a passive digital diary, Produco functions as an **Automated Life Coach**. It leverages a local-first browser database to analyze your daily inputs (Sleep, Study, Workout, Screen Time, Diet), find hidden behavioral patterns, and provide brutal, predictive feedback on your trajectory before consequences manifest.

---

## 📁 Project Architecture (Modular Vanilla Stack)

The application deliberately avoids heavy frameworks (like React or Vue) and backend cloud servers. It relies purely on the DOM, CSS3 hardware acceleration, and ES6+ JavaScript to ensure lightning-fast performance, zero latency, and absolute data privacy.

The codebase is split into a modular, maintainable structure:

* **`index.html` (The Skeleton):** Handles the DOM hierarchy, SVG icons, bottom-sheet overlays, and navigation routing.
* **`style.css` (The Paint):** Contains all dynamic CSS variables, Liquid Glass UI styling, keyframe animations, and zero-lag color gradients.
* **`script.js` (The Engine):** Houses the core state management, `localStorage` database wrapper functions, and the heavy algorithmic logic for the Intelligence Engines.
* **`sw.js` & `manifest.json` (The PWA):** Enables offline functionality, asset caching, and native mobile installation.

## 🧠 The Intelligence Engines (Predictive AI)

Produco shifts from *descriptive* tracking (what happened) to *predictive* tracking (what will happen) using three pure-JavaScript analytical algorithms.

### 1. The Consequence Engine (Predictive Warnings)
* **Objective:** Alert the user to behavioral consequences *before* they fail.
* **Mechanism:** A background `for` loop analyzes a 3-day sliding window of local data.
* **Logic Triggers:** * **Burnout Imminent (Critical):** If `sleep < 6 hours` for multiple days, the engine predicts a focus crash and dynamically injects a red warning card into the dashboard.
  * **Trajectory Warning (Warning):** If `screen time > study time` over a multi-day window, it flags a dopaminergic imbalance.
  * **Unstoppable Momentum (Positive):** Triggers when the user strings together consecutive days of high study hours and workouts.

### 2. The Life Score Algorithm
* **Objective:** Gamify daily existence into a single, digestible metric.
* **Mechanism:** Computes a real-time, weighted 0–100 score of the current day based on tracked habits.
* **The Mathematics:** * **Baseline:** Starts at 10 points.
  * **Heavy Weights:** Study time (+30 max) and Sleep (+30 max) drive the core score.
  * **Bonuses:** Workouts (+15) and perfect Diets (+15).
  * **The Penalty:** Subtracts 5 points for *every single hour* of screen time over a 2-hour threshold, easily destroying an otherwise perfect score.

### 3. The 7-Day Reality Check
* **Objective:** Automated, unsparing weekly accountability.
* **Mechanism:** Aggregates the past 7 days of inputs and compares internal ratios (e.g., total screen time vs. total study time).
* **Output:** Generates a brutally honest, context-aware paragraph summarizing the week, categorizing the user's trajectory as "Elite," "Average," or "Degrading."

## 🛠️ Core Tracking Modules

Produco is divided into highly specialized, interconnected modules accessible via the unified bottom navigation bar.

### 1. Month-Scoped Custom Diet Tracking
* **Feature:** A 10-item daily checklist for nutritional goals.
* **Architecture Security:** Users can edit their custom diet goals directly in the app. To protect historical data, these edits are saved *strictly* to the currently viewed month (`diet_cats_YYYY_M`). Changing a goal in August will not corrupt the data recorded in July.

### 2. Focus Auto-Tracker (Smart Surfacing)
* **Feature:** A built-in Pomodoro and stopwatch system for deep work.
* **Automation:** When a timer is stopped, the JavaScript automatically writes the tracked duration directly to the current day's lifestyle database. 
* **Smart Surfacing:** When the user opens the manual Lifestyle entry sheet, the UI automatically expands and pre-fills the tracked time to prevent double-entry and reduce friction.

### 3. Financial & Control Hub
* **Feature:** A dedicated ledger for tracking income, expenses, and college fee balances. 
* **Mechanism:** Aggregates localized data keys and renders a running balance. It also houses the Streak counter and Mood Tracker, analyzing consecutive days of positive `ls_YYYY_M_D` records.

## ✨ UI/UX Philosophy (Liquid Glass)

The interface is engineered to reduce cognitive load, utilizing behavioral psychology to ensure the user *wants* to log their data.

* **Progressive Disclosure:** Complex inputs (like inputting workout hours or screen time) are hidden by default via CSS (`display: none`). They only animate into view when the user physically flips a custom CSS toggle switch. This keeps the UI frictionless and visually clean on rest days.
* **Glassmorphism:** Bottom sheets and cards utilize `background: rgba(255,255,255,0.05)` combined with heavy `backdrop-filter: blur(16px)` to create a frosted glass effect that dynamically interacts with the underlying gradients.
* **Tactile Micro-Interactions:** Custom-built CSS toggle switches mimic native OS settings, paired with `triggerHaptic()` feedback calls to provide a micro-hit of dopamine upon task completion.

## 💾 Database Schema (Local-First)

The app requires zero internet connection to function. It uses the browser's native `localStorage` API, interfaced through custom `sv(key, value)` and `ld(key, fallback)` wrapper functions.

To ensure the app remains blazing fast over years of use, data is never clumped into a massive, slow-to-parse array. Every metric gets a unique, date-specific key:
* `ls_YYYY_M_D` → Stores a JSON object of the day's core metrics (sleep, study, workout, screen).
* `diet_YYYY_M_D_index` → Stores boolean values for specific nutritional items.
* `diet_cats_YYYY_M` → Stores the array of custom diet goals for that specific month.
* `tasks_YYYY_M_D` → Stores the daily agenda.

## 🚀 Developer Setup & Local Execution

Because Produco uses a Service Worker and a `manifest.json` file for its Progressive Web App (PWA) capabilities, opening the `index.html` file directly from your hard drive (`file:///`) will trigger CORS (Cross-Origin Resource Sharing) security errors in modern browsers.

**To run the app locally:**
1. Open the project folder in VS Code.
2. Install the **Live Server** extension (by Ritwick Dey).
3. Right-click `index.html` and select **"Open with Live Server"**.
4. The app will launch at `http://localhost:5500` with all service workers and PWA features fully enabled.
