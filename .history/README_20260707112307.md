# 🏎️ Produco 

A high-performance, proactive Life Operating System designed to track habits, predict burnout, and enforce accountability through intelligent data analysis and a premium "Liquid Glass" UI.

Produco is not just a digital diary; it is a predictive system. Built entirely with Vanilla web technologies, it leverages a local-first architecture to analyze daily inputs (Sleep, Study, Workout, Screen Time, Diet) and act as an automated Life Coach.

---

## 📁 Project Architecture (Modular Vanilla Stack)

The application is built without heavy frameworks (React/Vue) or backend servers, ensuring lightning-fast performance and offline capabilities.

* **`index.html` (The Skeleton):** Handles the DOM structure, bottom sheets, navigation routing, and layout grids.
* **`style.css` (The Paint):** Contains all CSS variables, animations, glassmorphism filters, and zero-lag CSS gradients.
* **`script.js` (The Engine):** Houses the `localStorage` database logic, mathematical algorithms, and predictive intelligence engines.
* **`sw.js` & `manifest.json` (The PWA):** Enables offline functionality, asset caching, and mobile installation.

---

## 🧠 The Intelligence Engines (Predictive AI)

Produco shifts from *descriptive* tracking (what happened) to *predictive* tracking (what will happen).

### 1. The Consequence Engine (Predictive Warnings)
* **How it works:** A background JavaScript loop analyzes a 3-day sliding window of your local data.
* **The Logic:** If `sleep < 6 hours` for multiple days, it calculates a focus crash and dynamically injects a critical "Burnout Imminent" card into the dashboard. If `screen time > study time`, it fires a "Trajectory Warning." It alerts you to consequences *before* they manifest.

### 2. The Life Score Algorithm
* **How it works:** Computes a real-time, weighted 0–100 score of your day.
* **The Logic:** It starts with a baseline of 10 points. It heavily weights Study (up to +30) and Sleep (up to +30), adds bonuses for Workouts (+15) and perfect Diets (+15), and applies a severe mathematical penalty for excess digital consumption (-5 points for every hour of screen time over 2 hours).

### 3. The 7-Day Reality Check
* **How it works:** An automated weekly review script.
* **The Logic:** It aggregates the past 7 days of inputs, compares ratios (e.g., total screen time vs. total study time), and generates a brutally honest, context-aware paragraph summarizing your week. It categorizes your trajectory as "Elite," "Average," or "Degrading."

## ✨ UI/UX Experience (Liquid Glass)

The interface is engineered to reduce cognitive load and mimic a native mobile OS.

* **Progressive Disclosure:** Complex inputs (like inputting workout hours) are hidden by default via CSS. They only animate into view when the user physically flips a custom CSS toggle switch, keeping the UI frictionless and clean on rest days.
* **Glassmorphism:** Bottom sheets and cards utilize `background: rgba(255,255,255,0.05)` combined with heavy `backdrop-filter: blur(16px)` to create a frosted glass effect that dynamically interacts with the underlying gradients.
* **Smart Surfacing:** If the built-in Pomodoro/Focus timer is used, the background script automatically writes the duration to the database. When opening the manual entry sheet, the UI automatically expands and pre-fills the tracked time to prevent double-entry.

## 💾 Database Architecture (Local-First)

Produco requires zero cloud servers and works entirely offline.

* **The Storage Medium:** Utilizes the browser's native `localStorage` API, interfaced through custom `sv(key, value)` and `ld(key, fallback)` wrapper functions.
* **Date-Scoped Keys:** Data is never clumped into a massive array. Every metric gets a date-specific key (e.g., `ls_2026_7_7` for daily stats, `diet_cats_2026_7` for monthly goals). This guarantees the database never slows down as years of data accumulate.
* **Month-Scoped Customization:** Users can edit their 10-item diet tracking goals. The architecture saves these custom lists *only* to the currently viewed month (`diet_cats_YYYY_M`). This ensures that altering goals mid-year does not corrupt historical tracking data.

## 🚀 How to Run (Development)

Because Produco uses a Service Worker and `manifest.json` for PWA capabilities, opening the `index.html` file directly from your hard drive (`file:///`) will trigger CORS security errors.

1. Open the project folder in VS Code.
2. Install the **Live Server** extension.
3. Right-click `index.html` and select **"Open with Live Server"**.
4. The app will launch locally (e.g., `http://localhost:5500`) with zero console errors.
